URL Rewriting

Pour fonctionner correctement, nous devons mettre en place une réécriture d'URL (URL rewriting)

Jusqu'à présent, pour accéder à une page PHP, il fallait écrire l'URL exacte du fichier PHP
(exemple : http://monsite.com/scripts/contact.php)

L'URL rewriting permet d'avoir des URLs plus propres, user-friendly et SEO-friendly
(exemple : http://monsite.com/contact)

URL Rewriting

Nous ne rentrerons pas dans les détails mais allons aborder les notions

L'URL rewriting permet de specifier des conditions et des actions à effectuer

Ainsi, http://monsite.com/page/2 peut avoir le même effet que http://monsite.com/index.php?page=2

Tout comme, http://monsite.com/category/anything peut avoir le même effet que http://monsite.com/une-page.php?category=anything

URL Rewriting

Avec Apache, le comportement à adopter par le serveur est spécifié dans un fichier .htaccess qu'il faudra placer dans le dossier concerné

On peut indiquer chacune des règles à suivre à l'intérieur

Les fichiers commençant par un "." n'apparaissent pas par défaut sur Mac. Pour les faire apparaître, ouvrez le terminal et tapez

defaults write com.apple.finder AppleShowAllFiles -bool true

Suivi de

killall Finder

URL Rewriting

  • Créez le fichier ".htaccess" dans le dossier "web"
  • Ajoutez-y le code ci-dessous
  • Remplacez "/path/to/app" par le chemin vers votre site
    Par exemple, pour http://localhost/monchemin/web/, le chemin est "/monchemin/web"

<IfModule mod_rewrite.c>
    Options -MultiViews

    RewriteEngine On
    RewriteBase /path/to/app
    RewriteCond %{REQUEST_FILENAME} !-f
    RewriteRule ^ index.php [QSA,L]
</IfModule>

URL Rewriting

Ce code provient de la documentation de Silex

Il active l'URL rewriting et permet de rediriger toutes les requêtes vers index.php

L'URL sera ensuite analysée par Silex pour pouvoir servir le bon contenu

3 - Créer les premières routes

Routes

Une route correspond au chemin vers une destination
(fonctionne en anglais)

La méthode $app->get() permet de définir une route

Dans l'exemple, si le visiteur accède à l'url http://localhost/monchemin/web/hello, la fonction de la route sera executée

Routes

Avec Silex, il est possible de spécifier des routes avec des paramètres


$app->get('/hello/{name}', function($name) {
    return 'Hello '.$name;
});

exemple : http://localhost/monchemin/web/hello/toto

Le nom du paramètre est défini entre accolades

Il faut ensuite rajouter la variable avec l'exacte même nom dans les paramètres de la fonction pour pouvoir l'utiliser dans la fonction

Routes

Il est, bien sûr, possible de spécifier des routes avec plusieurs paramètres


$app->get('/hello/{first_name}-{last_name}', function($first_name,$last_name) {
    return 'Hello '.$first_name.' '.$last_name;
});

Routes

L'ordre des routes a son importance

La première route qui "match" la requête sera utilisée

Dans l'exemple suivant, la deuxième route ne sera jamais utilisée car la première est prioritaire


// Route 1
$app->get('/hello/{name}', function($name) {
    return 'Route 1';
});

// Route 2
$app->get('/hello/tata', function() {
    return 'Route 2';
});

Routes

Il est possible de spécifier le format attendu du paramètre


$app->get('/page/{id}', function($id) {
    return 'Page '.$id;
})
->assert('id','\d+');

\d+ signifie "1 ou plusieurs chiffre"
C'est ce qu'on appelle une regex

Routes

Nous n'allons pas apprendre à faire des regex mais en voici quelques-unes bien utiles :

  • \w+ (un mot)
  • \d+ (un nombre)
  • [a-z0-9_-]{3,16} (pseudo entre 3 et 16 caractères)
  • [a-z0-9-]+ (un slug)
  • ([a-z0-9_\.-]+)@([\da-z\.-]+)\.([a-z\.]{2,6}) (une adresse mail)

La méthode "__construct()" est appelée à l'initialisation "new Snippets_Model($pdo)"

  • Dans __construct() enregistrez $pdo dans les propriétés de la classe 
    
function __construct($pdo)
{
    $this->pdo = $pdo;
}
  • Complétez la méthode get_by_page() afin quelle "return" les snippets pour telle page
  • Complétez la méthode get_by_category_slug() afin quelle "return" les snippets pour tel slug de categorie
  • Complétez la méthode get() qui va simplement appeler get_by_page() avec les paramètres par défaut

snippets.class.php


<?php

/**
* Snippets Model
*/
class Snippets_Model
{

    function __construct($pdo)
    {
        $this->pdo = $pdo;
    }

    function get()
    {
        // Return first page snippets (default)
        return $this->get_by_page();
    }

    function get_by_page($page = 1, $limit = 4)
    {
        // Limit values
        $page -= 1;
        $start = $page * $limit;

        // Sql
        $prepare = $this->pdo->prepare('
            SELECT
                s.title,
                s.content,
                c.title AS category_title,
                c.slug AS category_slug
            FROM
                snippets AS s
            LEFT JOIN
                categories AS c
            ON
                s.id_category = c.id
            LIMIT
                :start,:limit
        ');

        // Bind values
        $prepare->bindValue('start',$start,PDO::PARAM_INT);
        $prepare->bindValue('limit',$limit,PDO::PARAM_INT);

        // Execute / Fetch
        $prepare->execute();
        $snippets = $prepare->fetchAll();

        // Return
        return $snippets;
    }

    function get_by_category_slug($slug)
    {
        // Sql
        $prepare = $this->pdo->prepare('
            SELECT
                s.title,
                s.content,
                c.title AS category_title,
                c.slug AS category_slug
            FROM
                snippets AS s
            LEFT JOIN
                categories AS c
            ON
                s.id_category = c.id
            WHERE
                c.slug = :slug
        ');

        // Bind slug value
        $prepare->bindValue('slug',$slug);

        // Execute / Fetch
        $prepare->execute();
        $snippets = $prepare->fetchAll();

        // Return
        return $snippets;
    }
}

  • Complétez-le de manière à afficher le titre du snippet, son contenu et sa catégorie en lien vers la page de catégorie

Celle-ci doit apparaître dans les pages "home" et "page"

Nous allons donc pouvoir créer un include twig

snippets.class.php


    /* ... */

    function get_pages($current = 1, $limit = 4)
    {
        $query  = $this->pdo->query('SELECT COUNT(*) AS count FROM snippets');
        $result = $query->fetch();
        $count  = ceil($result['count'] / $limit);
        $pages  = array();

        for($i = 1; $i <= $count; $i++)
        {
            $pages[] = array(
                'number'  => $i,
                'current' => $current == $i
            );
        }

        return $pages;
    }
}

6 - Gérer les erreurs

Lorsqu'une erreur se produit, Silex va essayer de la catcher et de renvoyer un message propre comme "Sorry, the page you are looking for could not be found."

Il est possible d'afficher plus d'information.
Pour cela, il suffit de passer l'élément debug de $app sur true


$app = new Silex\Application();
$app['debug'] = true;

Il est important de penser à le repasser sur false une fois le site en production

Il est aussi possible d'utiliser ses propres pages d'erreurs

Cela fonctionne comme la création d'une route sauf qu'il faut écrire error au lieu de get et il n'y a pas à préciser de route (logique)

Il est possible de récupérer l'exception ainsi que le code d'erreur dans les paramètres de la fonction


// Errors
$app->error(function (\Exception $e, $code)
{
    // Globals
    global $app;

    $data = array(
        'title' => $code
    );
    if($code == 404)
        return $app['twig']->render('404.twig',$data);
});
  • Créez la route d'erreur
  • Créez un fichier 404.twig dans le dossier view/
  • Mettez-y ce que vous souhaitez ("Page not found")
  • Testez une URL aléatoire qui n'existe pas
